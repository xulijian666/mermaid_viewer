## 2. 开发模式启动（推荐日常使用）

### 2.1 仅看前端界面（最快）
在项目根目录执行：

```powershell
npm run dev
```

说明：
- 启动 Vite 本地开发服务器（通常是 `http://localhost:5173/`）。
- 适合调样式、调交互，不涉及 Tauri/Rust 能力时首选。

### 2.2 桌面联调模式（含 Tauri 能力）
在项目根目录执行：

```powershell
npm run tauri:dev
```

说明：
- 会启动前端 + Tauri 桌面进程。
- 首次或 Rust 变更后会看到 `Compiling ...`，属于正常编译过程。
- 适合联调导出、系统对话框等桌面能力。

---

## 3. 打包为 exe（Windows）

在项目根目录执行：

```powershell
npm run tauri:build
```

说明：
- 该命令会触发正式构建并生成 Windows 安装包/可执行产物。
- 打包完成后，产物通常位于：
  - `src-tauri\target\release\bundle\`
- 常见会看到的子目录（按配置可能略有不同）：
  - `nsis\`（安装程序，常见为 `.exe`）
  - `msi\`（MSI 安装包）

---

## 4. 常见说明

- 开发阶段建议优先使用 `npm run dev`，速度快，反馈及时。
- 需要验证桌面能力时再使用 `npm run tauri:dev`。
- 需要给测试或客户安装时再执行 `npm run tauri:build` 生成 exe。
